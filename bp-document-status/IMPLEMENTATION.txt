Внедрение: статус документа и блокировка (1С:Бухгалтерия)
=========================================================

Каталог поставки: bp-document-status/
Область: документы РеализацияТоваровУслуг и ПоступлениеТоваровУслуг.

Порядок работ в Конфигураторе / EDT
------------------------------------
1. Перечисление — см. metadata/01_Enumeration.txt
2. Реквизит СтатусДокумента, роль, общий модуль — см. metadata/02_Documents_And_Role.txt
3. Создайте общий модуль ДокументыСтатусыОпераций (сервер, без привилегированного режима)
   и скопируйте код из src/CommonModules/ДокументыСтатусыОпераций/Module.bsl
4. Для каждого документа (РеализацияТоваровУслуг и ПоступлениеТоваровУслуг):
   • добавьте реквизит шапки СтатусДокумента;
   • в модуль объекта — вызовы из snippets/<ИмяДокумента>/DocumentObject_Module_snippet.bsl
     или snippets/DocumentObject_Module_snippet.bsl (Реализация);
     в ПередЗаписью передавайте РежимЗаписи четвёртым параметром; не дублируйте типовые процедуры;
   • в модуль основной формы — фрагменты из snippets/<ИмяДокумента>/Form_Module_snippet.bsl
     (для Поступление — snippets/ПоступлениеТоваровУслуг/);
     в конце ПриСозданииНаСервере и ПриЧтенииНаСервере — бпСтатус_ПриСозданииИПриЧтенииНаСервере();
     в конце ПослеЗаписиНаСервере — бпСтатус_ПослеЗаписиНаСервере();
   • в конструкторе формы: реквизит формы бпСтатусДокумента + поле ввода;
     обработчик ПриИзменении — бпСтатус_СтатусДокументаПриИзменении (клиент).
   Расширение: *Extension_snippet.bsl из той же папки, что и документ.
5. Роль ИзменениеСтатусаОперационныхДокументов назначьте только выбранным пользователям бухгалтерии.
6. Обмен с КА — см. exchange-bp-ka.txt
7. Прогон тестов — manual-tests.txt

Зависимости
-----------
• Имена перечисления СтатусыДокументовОпераций и значений ВРаботе / ВДоработке / Выполнено
  должны совпадать с кодом общего модуля.
• Имя роли в метаданных должно совпадать с функцией ИмяРолиИзменениеСтатуса() в модуле
  (или измените функцию под своё имя роли).

Расширение конфигурации
-----------------------
• Предпочтительно вносить изменения через расширение, если политика поддержки это допускает.
• Пошагово только расширение: metadata/03_Extension_Only.txt и сниппеты *Extension* в snippets/.
• При дублировании типовых процедур ПередЗаписью используйте аннотации &Вместо / &После
  согласно документации платформы по расширениям.
