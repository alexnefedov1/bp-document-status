Внедрение только через расширение конфигурации (1С:БП КОРП 3.0 и аналоги)
Область: РеализацияТоваровУслуг и ПоступлениеТоваровУслуг.
================================================================================

Общие правила платформы
-----------------------
• В расширении добавляются свои объекты: перечисление, роль, общий модуль, реквизиты документов, расширения форм и модулей.
• Типовой код не копируют целиком: в модули расширения добавляют **отдельные** процедуры с аннотациями **&После(...)** или **&Вместо(...)** (реже), вызывающие общий модуль расширения.
• Сигнатура процедуры с **&После** должна **совпадать** с сигнатурой типовой процедуры, к которой «цепляется» расширение (проверьте в основной конфигурации в EDT/Конфигураторе).
• Целевые документы: **РеализацияТоваровУслуг**, **ПоступлениеТоваровУслуг** (строка в вызовах — как в дереве метаданных).

Порядок работ (расширение)
--------------------------
1. Создайте расширение конфигурации (префикс имён по политике ИБ), подключите к базе, обновите конфигурацию БД.

2. Перечисление **СтатусыДокументовОпераций** — в расширении, по образцу metadata/01_Enumeration.txt.

3. Роль **ИзменениеСтатусаОперационныхДокументов** — в расширении, по metadata/02_Documents_And_Role.txt  
   (в профиль группы доступа добавьте роль + права на перечисление и документы, см. тот же файл).

4. Общий модуль **ДокументыСтатусыОпераций** — только в расширении: серверный, не привилегированный;  
   скопируйте код из src/CommonModules/ДокументыСтатусыОпераций/Module.bsl.

5. Для **Документ.РеализацияТоваровУслуг** и **Документ.ПоступлениеТоваровУслуг** в расширении:
   • реквизит шапки **СтатусДокумента** (тип — перечисление из п.2);
   • **модуль объекта расширения** — DocumentObject_Module_Extension_snippet.bsl из папки документа
     (snippets/ или snippets/ПоступлениеТоваровУслуг/; при необходимости поправьте **&После** под типовую процедуру);
   • **расширение основной формы** — Form_Module_snippet.bsl + Form_Module_Extension_snippet.bsl из той же папки;
   • в конструкторе формы:
     — реквизит формы **бпСтатусДокумента** (перечисление) + поле ввода с путём **бпСтатусДокумента** (основное для проведённых документов);
     — опционально **Объект.СтатусДокумента** (не **Объект.Ссылка.СтатусДокумента**);
     — одно видимое поле статуса (не дублировать элемент 7 раз);
     **ПриИзменении** → бпСтатус_СтатусДокументаПриИзменении (клиент).
   • если роль есть, а поле серое: кнопка «Проверка прав» → в сообщениях «Найдено полей…» должно быть ≥ 1;
     иначе элемент на форме не привязан к реквизиту (см. диагностику в общем модуле).

6. Назначьте роль пользователям через профили и группы доступа БП (см. переписку и metadata/02_Documents_And_Role.txt).

7. Прогон — manual-tests.txt.

Куда вставлять код (частая ошибка)
----------------------------------
  Модуль объекта расширения     → только DocumentObject_Module_Extension_snippet.bsl
  Модуль формы расширения       → Form_Module_snippet.bsl + Form_Module_Extension_snippet.bsl

Если **бпСтатус_ЗаписатьСтатусИзБазыВРеквизитФормыОбъект** лежит в модуле объекта — конфигуратор
выдаст «РеквизитФормыВЗначение не определена» (как в служебных сообщениях Ugse_w).

Важно про форму
---------------
• Методы **РеквизитФормыВЗначение** / **ЗначениеВРеквизитФормы** — только **модуль формы** (Form_Module_snippet.bsl).
• Если основной реквизит формы не называется **Объект** — замените **"Объект"** в бпСтатус_ЗаписатьСтатусИзБазыВРеквизитФормыОбъект.

Важно про модуль объекта
-------------------------
• **&После("ПередЗаписью")** — типовая сигнатура в БП чаще всего **ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)**; иначе скорректируйте параметры процедуры расширения под вашу версию.
• **&После("ОбработкаЗаполнения")** — часто **ОбработкаЗаполнения(ДанныеЗаполнения, СтандартнаяОбработка)**.

Совместимость и поддержка
-------------------------
• Перед сдачей в эксплуатацию согласуйте политику **расширений** с владельцем конфигурации (обновления БП могут менять имена/сигнатуры типовых процедур — расширение нужно перепроверять после обновления).

Файлы-сниппеты для расширения
-----------------------------
• snippets/DocumentObject_Module_Extension_snippet.bsl — модуль объекта документа (&После).
• snippets/Form_Module_Extension_snippet.bsl — модуль формы (&После к типовым серверным процедурам).
