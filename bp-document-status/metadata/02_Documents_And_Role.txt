Реквизит шапки документа
========================

Имя реквизита: СтатусДокумента
Тип: ПеречислениеСсылка.СтатусыДокументовОпераций
Обязательное: Да
Заполнение по умолчанию: через ОбработкаЗаполнения (ВРаботе)

Добавить в документы (проверьте имена в дереве метаданных вашей БП):

  • РеализацияТоваровУслуг
  • ПоступлениеТоваровУслуг

Расширение формы: основная форма каждого документа (типовая «ФормаДокумента»).

Роль
====

Имя роли (англ.): ИзменениеСтатусаОперационныхДокументов
Синоним: Изменение статуса операционных документов

Права роли (минимально):
  • Чтение и Просмотр для документов РеализацияТоваровУслуг и ПоступлениеТоваровУслуг (если бухгалтерам нужно открывать)
  • Плюс те же права, что нужны для записи документа при смене только статуса —
    обычно достаточно типовых прав «добавление/изменение» по этим документам,
    которые уже есть у бухгалтерии; новая роль добавляет только «логику» в коде.

Доступ к данным статуса (иначе поле пустое, список перечисления не раскрывается, ввод заблокирован платформой):
  • Перечисление СтатусыДокументовОпераций — у профиля группы доступа должны быть как минимум Чтение (и Просмотр, если разделяется в вашей версии БП) у всех пользователей, которые открывают документы со статусом.
  • У документов с реквизитом СтатусДокумента — в профиле должно быть право Чтения (и Изменения при смене статуса) на этот реквизит; при ограничении на уровне записей пользователь должен попадать в область видимости строки документа (та же организация / вид доступа и т.д.).

Важно: новая роль не заменяет профиль бухгалтера — её добавляют к пользователям,
которым разрешено менять статус (см. план).

Общий модуль
============

Создать общий модуль: ДокументыСтатусыОпераций
  • Сервер: Да
  • Вызов сервера: Нет (вызов только с сервера формы и модуля объекта)
  • Привилегированный: Нет (проверка роли должна быть в пользовательском контексте)
