Ручные сценарии проверки (после внедрения в тестовой базе)
============================================================

Подготовка
----------
• Создать пользователя A без роли ИзменениеСтатусаОперационныхДокументов.
• Создать пользователя B с этой ролью (+ обычные права на документы).
• Сценарии 1–11 — на РеализацияТоваровУслуг; повторите 1–9 на ПоступлениеТоваровУслуг (основная форма).

Смена статуса и роль
--------------------
1. Под A: открыть новый документ — статус по умолчанию «В работе», поле статуса недоступно (серое / без ввода).
2. Под A: попытаться сменить статус в поле — значение сразу возвращается к «В работе», сообщение о роли; запись с другим статусом — отказ.
3. Под B: сменить статус на «В доработке», записать — успех.
4. Под B: установить «Выполнено», записать — успех; остальные поля формы недоступны, статус доступен.
5. Под A: открыть тот же документ — форма только просмотр; попытка записи через обработку — отказ.
6. Под B: сменить «Выполнено» на «В работе» — успех; редактирование шапки/ТЧ снова доступно.

Блокировка содержимого при «Выполнено»
---------------------------------------
7. Под B: выставить «Выполнено», записать.
8. Под B: попытаться изменить контрагента/сумму/строку ТЧ (не статус) — при записи отказ с текстом про «только статус».

Удаление
--------
9. Документ в «Выполнено»: под любым пользователем попытка удаления — отказ (сначала сменить статус под B).

Обмен БП ↔ КА
-------------
10. В БП: документ «В доработке»; приём изменений из КА по тому же объекту — успех;
    СтатусДокумента в БП не перезаписан из КА (остался из БП или «В работе» для нового).
11. В БП: документ «Выполнено»; приём данных из КА по этому объекту — отказ, документ в БП без изменений.
12. В БП: документ «Выполнено»; изменить только что-то в КА и убедиться, что выгрузка из БП этого документа
    не выполняется (после записи в БП без смены статуса — нет регистрации в обмен, см. журнал/очередь).
13. Снять «Выполнено» под пользователем B — обмен снова допускается.
14. Новый документ из КА: статус в БП = «В работе», не пусто.

Примечания
----------
• Если типовая форма переопределяет ТолькоПросмотр в своих обработчиках — проверить порядок вызовов
  (возможно, вызывать бпСтатус_НастроитьФорму() после типовой логики).
• Сравнение табличных частей идёт построчно в порядке выгрузки; при смене порядка строк без изменения
  данных теоретически возможна ложная несовпадаемость — при необходимости доработать сравнение ТЧ.
